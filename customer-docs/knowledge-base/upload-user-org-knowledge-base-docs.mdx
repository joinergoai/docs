Upload knowledge base documents when Ergo needs reusable context such as playbooks, enablement docs, FAQs, or policies.

## Supported upload behavior

- The checked backend upload route accepts PDF, DOCX, DOC, TXT, and Markdown files.
- Files must be 25 MB or smaller.
- The upload route extracts text, stores a document record, uploads the original file when storage succeeds, and indexes extracted text for search.
- Document status is `pending`, `indexed`, or `failed`.

## Upload a document

- Open the knowledge base document manager.
- Select the document area or drag in a supported file.
- Wait for the document to finish processing.
- Confirm the document reaches `indexed` before expecting Ergo to use it as context.

## Scope note

Some UI surfaces show user and org document areas. In the checked upload route, uploaded knowledge base files are stored against the uploading user and org, and the public docs should avoid promising org-wide availability unless your current workspace UI explicitly grants it.
