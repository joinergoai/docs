Post-call reference documents are admin-managed files used as drafting context for post-call email generation.

## What admins can do

- Upload reference documents from post-call configuration.
- Keep up to the configured document limit.
- Delete reference documents that should no longer guide drafts.
- Edit a generated summary.
- Retry summary generation when a summary fails.

## Summary states

- **Pending**: Ergo is generating the document summary.
- **Ready**: the summary is available for post-call context.
- **Failed**: summary generation failed and can be retried where the UI exposes retry.

## How summaries are used

- Ergo uses the summary as lightweight drafting context.
- Full document content may be pulled when needed by the workflow.
- Reference docs are post-call drafting guidance, not a replacement for reviewing the generated draft.
