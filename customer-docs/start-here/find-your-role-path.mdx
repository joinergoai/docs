Use this page when you know the person's job but not the right product area or setup article yet. The role paths explain what each person can get out of Ergo, then point to the deeper setup, workflow, troubleshooting, and reference docs.

## Choose a path

- [RevOps playbook](/customer-docs/roles/revops): keep pipeline data trustworthy, configure CRM writeback, and reduce audit work.
- [Sales manager playbook](/customer-docs/roles/sales-manager): review pipeline risk, team follow-up, coaching evidence, reporting, and adoption.
- [AE playbook](/customer-docs/roles/ae): know what to work on, ask deal questions, generate reviewable follow-up drafts, and keep mapped CRM fields updated when configured.
- [CSM playbook](/customer-docs/roles/csm): see account history, find renewal or expansion signals, and draft customer outreach.
- [SDR playbook](/customer-docs/roles/sdr): prioritize outreach, prepare handoffs, draft follow-up, and use deal context.

## If you are still not sure

- Start with [Welcome to Ergo](./welcome-to-ergo) if you need the basic product model.
- Use [Navigating Ergo](./navigating-ergo) if you know the workflow but not where it lives.
- Use [Roles and permissions](./roles-and-permissions) if the issue is access or visibility.
- Use [Troubleshooting](/customer-docs/troubleshooting/index) if something is missing, failed, duplicated, or blocked.
