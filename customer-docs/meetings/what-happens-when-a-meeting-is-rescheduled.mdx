Rescheduling changes the calendar event first. The notetaker state updates only after Ergo receives the changed calendar event and the notetaker provider reflects the new state, so calls changed close to start time need extra review.

## What can change

- Meeting time.
- Meeting link.
- Calendar event ID or recurring instance ID.
- Attendees.
- Whether the meeting still qualifies for bot scheduling.

## What to do

- Recheck the event on the dashboard after the calendar update syncs.
- If the meeting link changed, confirm the upcoming card shows the new URL before the call starts.
- If the notetaker is still tied to the old link, cancel and schedule it again or use the update-link flow when the UI exposes it.
- If the meeting was rescheduled very close to start time or the bot already joined at the old time, use [Add the bot to a live or ad hoc meeting](./add-the-bot-to-a-live-or-ad-hoc-meeting) to dispatch a fresh bot.
- For recurring meetings, choose whether the change affects only one occurrence or future occurrences.

## What to expect

- A changed calendar event may briefly show stale notetaker state.
- If an old event was explicitly unscheduled, Ergo may keep a marker so it does not re-add the bot automatically.
- If the meeting starts before the updated event is available, use [Add the bot to a live or ad hoc meeting](./add-the-bot-to-a-live-or-ad-hoc-meeting).
- Avoid editing or rebooking an event while the bot is already live unless you can verify the bot remains in the meeting and the completed record processes normally.
