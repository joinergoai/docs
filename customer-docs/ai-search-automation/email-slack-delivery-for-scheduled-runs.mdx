Scheduled run delivery can send results to email recipients, Slack destinations, or multiple destinations when configured.

## Who can use this

- Users configuring scheduled run recipients.
- Admins troubleshooting failed delivery.

## Before you start

- Confirm the run prompt is ready.
- Connect Slack before selecting Slack delivery.
- Confirm email recipients and Slack channels or DMs are appropriate for the run output.

## Steps

- Open the scheduled run.
- Add email recipients for email delivery.
- Add a Slack channel or DM destination when Slack delivery is available.
- Save the run.
- Use **Run** to test delivery.
- Check history for delivery result details.

## What to expect

- Delivery destinations can include email and Slack.
- Slack delivery is sent through the connected account/workspace configured for the user or org.
- Execution history can show delivery result details such as email recipients, Resend ID, Slack draft ID, Slack permalink, channel ID, or channel name when available.
- Delivery success can differ from agent execution success, so check both the run status and destination result.

## Common issues

- Slack is disconnected or the destination is no longer accessible.
- An email recipient is invalid.
- The run succeeds but one destination fails.
- The output includes sensitive context and should not be delivered to a broad channel.
