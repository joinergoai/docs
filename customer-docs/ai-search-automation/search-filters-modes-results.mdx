Filters and modes change what Search asks for. Use them when broad results are noisy or missing the source type you care about.

## Who can use this

- Users searching across multiple source types.
- Managers narrowing results to a specific team member's meetings.

## Before you start

- Know whether you want meetings, email threads, or documents.
- Decide whether the result should be tied to a specific attendee.
- Start with broad search before adding multiple filters.

## Steps

- Enter a query of at least two characters.
- Choose a source filter: meeting, email thread, or user document.
- Choose a search mode: hybrid, keyword, or semantic.
- Use the member filter when you want records tied to a specific attendee.
- Change page size or move between pages when more results are available.
- Remove filters if the result set is too narrow.

## What to expect

- **Hybrid** is the default search mode.
- **Keyword** favors direct term matching.
- **Semantic** favors meaning-based similarity.
- Selecting an attendee filter clears the entity filter in the current UI.
- Error and no-result states can mean the query, filters, access, or source sync needs adjustment.

## Common issues

- Too many filters hide relevant results.
- Keyword mode misses paraphrased content.
- Semantic mode returns related content that does not contain the exact term.
- The selected attendee is not associated with the record.
- The page offset points past the available results after a filter change.
