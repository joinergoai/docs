Picklist options and numerical buckets control how Reporting groups field values. Use them to make charts easier to compare across teams and time periods.

## Picklist fields

Picklist fields use configured options. You can:

- Add options one at a time.
- Paste comma-separated or line-separated options.
- Merge new options with existing options.
- Update a field's picklist values.
- Group selected picklist values into user-configured categories when the field supports it.

Keep option labels mutually understandable. Reporting is easier to read when managers do not need to remember subtle differences between nearly identical values.

## Numerical fields

Numerical fields use named ranges. When configuring ranges:

- Each range needs a minimum and maximum.
- The minimum must be lower than the maximum.
- Ranges should not overlap.
- Ranges are re-indexed after removal.

Numerical filters use the configured range values. A selected range becomes a numeric query against that field.

## After changing buckets

- Refresh the dashboard or chart.
- Check whether the chart is using cached data.
- Re-run or request updated clustering if the field's grouping logic changed materially.
- Revisit saved charts and reporting email cadences that rely on the field.

Changing bucket configuration changes how future reads group the field; it does not prove that every historical source has been reprocessed.
