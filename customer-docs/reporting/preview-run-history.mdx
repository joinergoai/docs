Run history shows what happened when a Reporting email cadence tried to generate or send a report.

## Open run history

1. Open **Reporting**.
2. Select **Email Cadences**.
3. Choose the cadence.
4. Open the run history or a recent run.

The run detail drawer shows lifecycle timestamps, run status, trigger, mode, cadence snapshot, selected fields, generated output, chart results, and email delivery details when available.

## Run statuses

Runs can be:

- Queued
- Running
- Succeeded
- Failed
- Skipped

Active runs are refreshed periodically in the cadence list. A run may be queued or running before the final status is known.

## Triggers and modes

- **Scheduled**: created by the cadence schedule.
- **Manual test**: queued from a test run.
- **Manual send**: queued from send now.
- **Preview mode**: renders the report without normal scheduled delivery.
- **Send mode**: prepares and sends to recipients when the run succeeds.

## Failure stages

Failure details may identify a stage such as preflight, queue, agent analysis, chart data, chart render, email render, email send, or persistence. Use the stage to decide whether to adjust the cadence, check source data, retry later, or contact support.

## Diagnose missing output

- Confirm the cadence is enabled if you expected a scheduled run.
- Check whether the latest run is still queued or running.
- Review selected fields or auto-selected chart fields.
- Check recipient count and email result.
- If analysis failed, simplify instructions or verify the selected fields have enough recent data.
- If chart rendering or email sending failed, retry after checking for known platform issues.
