Custom reporting fields define what Ergo can analyze and chart. Configure them carefully; unclear fields create unclear dashboards.

## Who can use this

- Users with Reporting create access can manage custom fields.
- Fields can be organization-level or team-level depending on where you add them.

## Field types

- **Sentence**: captures themes or excerpts from customer conversations.
- **Picklist**: groups results into configured options.
- **Numerical**: groups numeric values into configured ranges.

Each field also has a dialogue focus: customer, sales rep, or both. Use the focus to tell Ergo whose language matters for that field.

## Create a field

1. Open **Reporting**.
2. Select **Custom Fields**.
3. Choose whether to add the field at the organization level or a team section.
4. Enter a field name.
5. Choose the field type.
6. Write a clear description.
7. Add picklist options if the type is picklist.
8. Choose dialogue focus.

The field autosaves after required inputs are present. If autosave fails, retry before using the field in reports.

## Use AI suggestions

AI-assisted field generation can draft a field name, type, description, focus, and picklist options. Review the suggestion before saving it. Generated configuration should be treated as a draft, not a final operating definition.

## Keep fields clean

- Avoid duplicate field names.
- Do not use periods in field names.
- Prefer stable names because saved charts and email cadences reference fields.
- Delete fields only after checking dashboards, cadences, and workflows that may depend on them.
- Add default reporting fields when you want a verified starting set instead of writing every field manually.

## Data freshness

New or changed fields may not immediately show useful data. Reporting depends on source data, processing, clustering, and any field-specific refresh or backfill work. If a field is important for historical reporting, confirm how historical data will be populated before promising results to the team.
