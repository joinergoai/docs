Use this checklist before depending on Ergo Desktop for important meetings.

## Who can use this

- New Desktop users.
- Admins and operators helping a team roll out local recording.

## Before you start

- Confirm Desktop access is enabled for the workspace.
- Install the current app from the approved download path.
- Decide whether users should use Desktop, the cloud notetaker, or both.

## Steps

- Install Ergo Desktop.
- Sign in through the browser tab opened by Ergo Desktop and confirm the authenticated email.
- Grant macOS permissions for Ergo Desktop, not only browser permissions, then quit and reopen the app.
- Connect or enable Ergo Desktop Notetaker if your team uses the integration card.
- Run a short test recording.
- Stop the recording and wait for upload and processing.
- Confirm the meeting appears in Ergo with the expected title, attendees, and transcript/recording status.

## What to expect

- A successful test should confirm capture, upload, and backend processing.
- Calendar-sourced recordings, detected meetings, and Quick Note recordings may carry different metadata.
- If a cloud bot is also scheduled, duplicate-source handling should be checked before broad rollout.

## Common issues

- Desktop access is not enabled, so download or setup is blocked.
- The app is signed in as the wrong user.
- Permissions are granted but the app was not quit and reopened.
- The recording button does not respond; quit Ergo Desktop completely and reopen it before retesting.
- The recording captured locally but failed upload or processing.
