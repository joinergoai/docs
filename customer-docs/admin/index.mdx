Use this section to find the Admin article that matches the workflow, access change, or setup issue in front of you.

## Who this section is for

- Admins managing teams, members, roles, access, defaults, and organization settings.
- Super Admins managing organization-wide controls such as global meeting access, persona, pipeline access, usage, and impersonation.

## Before you start

- Sign in with admin access or delegated permission for the area you are changing.
- Confirm the team, member, workspace, and downstream access impact before saving.
- Use super-admin-only controls only when the change should affect the broader organization.
- Admin pages are available from the Admin navigation item for admin users. Several controls are visible only to Super Admins.
