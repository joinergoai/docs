Use this page when onboarding or a setup step appears incomplete after the user has tried to connect sources or save workspace settings.

## Symptoms

- Onboarding cannot advance to the next step.
- A setup step saves but later appears incomplete.
- A connected source, workspace setting, notetaker, CRM field, or reporting default is missing.
- The user skipped onboarding and later finds core workflows are not running.

## Most common causes

- The user's role-specific setup path has not been completed.
- A required CRM, email, calendar, collaboration, notetaker, or reporting step is missing.
- The connected account does not have the permission or scopes needed for the workflow.
- Setup was completed for one user, but another user still needs their own source connections.

## What to check

- Follow the setup checklist in role order instead of jumping between steps.
- Confirm each source shows connected after OAuth completes.
- Ask an admin or Super Admin to complete workspace-level settings when the user cannot see them.
- Check whether the user is a standard user, admin, Super Admin, or spectator before changing access.

## Resolution steps

- Reconnect the source only when the setup step shows disconnected, expired, or incomplete.
- For CRM setup, confirm field mapping, stages, and property permissions before expecting updates.
- For meeting setup, confirm calendar/notetaker state and retry with one upcoming meeting.
- For reporting setup, confirm dashboard creation and reporting access separately.

## When to contact support

- Contact support when the expected role, connected source, and setup step are confirmed but the issue repeats.
- Include the workspace, affected user, setup step, source system, visible error, and approximate time window.
- Keep examples generalized and avoid sharing sensitive customer content in support requests.
